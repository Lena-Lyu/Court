# Court — AI Agent 的溯源优先记忆系统

> **不做仓库，做法庭。**
>
> 生肉为唯一权威。信念现造即弃。没有出处的东西不被信任。一切可从零开始。

[English version](README.md)

## 问题

AI agent 会忘。不是因为模型不够聪明——是因为**记忆系统把存储当成了真相。**

Mem0、Supermemory、Anthropic 的 Dreaming——它们都在做同一件事：写入时把对话消化成"事实"，存起来当真相，信任它直到它腐烂。三种看起来不同的失败，同一个根源：

| 失败 | 表现 | 根源 |
|------|------|------|
| **幻觉** | 模型无中生有 | 信念丢失了出处 |
| **投毒** | 脏数据污染系统 | 信念的出处是伪造的 |
| **腐烂** | 旧信息永远是"对的" | 信念活过了它的出处 |

**根源永远是：provenance（出处）的丢失。** 一条信念脱离了它的来源，被当成事实而非主张。

---

## 洞察

LLM 本质是 stateless 函数——每次推理都是一次新生。所谓的"记忆"不在模型权重里，在模型外面的 harness 里。真正的问题不是"怎么存更多"，而是**"怎么存，才能让每条信念都能被验证、被证伪、发现错了就被丢弃。"**

仓库：把什么都存进去，永远信任。法庭：存原始证据，用时当庭裁决，**判决从不归档为真理。** Court 就是那个法庭。

---

## 五条脊柱

每个组件——从存储到检索到 agent 间通信——遵循五条原则：

1. **出处是承重的。** 投毒 = 伪造出处。幻觉 = 丢失出处。腐烂 = 出处过期。

2. **生肉是唯一权威。** 原始观察，逐字记录。只被信任为"记录"，不是"真相"。

3. **信念现造即弃。** 没有常驻信念。每条主张在使用的当下裁决，带置信度和出处，用完即丢。

4. **越往深层写越危险。** 可撤销性梯度：原文 > KV cache > 隐空间 > 权重。压缩能力（模型会什么），外置事实（模型声称什么）。

5. **拿不到出处时，用证伪代替可读性。** 读不懂里面是什么？试着证明它错了。同一套索引，反过来问："有没有证据和它矛盾？"

## 三层架构

```
L0 — 生肉层（SQLite，append-only，SHA256 内容寻址）
    "用户说：'我改用 Rust 了'"——不是"用户偏好 Rust"

L1 — 可重算索引（10 通道 RRF 融合，零 LLM 检索）
    只存指针——"路标，不是真相。" 可随时删除并从 L0 重建。

L3 — 用时裁决（法庭）
    1. 多通道召回（毫秒级，零 LLM）
    2. 路径积分：独立佐证 × 出处强度 × 新鲜度 × 内部一致性
    3. 证伪搜索：同一套索引，相反的提问
    4. 查锚：每条主张必须能指回 L0——找不到锚 = 标为疑似幻觉
    5. 定罪：常规情况本地小模型结案，高风险才升级云端 SOTA
    6. 用完即弃：判决作为新观察记回 L0，然后丢弃
```

**理论基础：** [Why Storage-Time Compression Fails](docs/dpm-theory.pdf) — 用 rate-distortion theory、time-bounded epiplexity 和信息瓶颈方法，从信息论角度分析为什么存原文、查询时按需提取是结构最优的，而非只是偏好。

## Agent 层 + Weaver 监督者

**Agent 铁律：** 行动前，先把要依赖的关键信念过一遍法庭。

**Weaver：** 正交监督层，硬边界执行。利用 KV cache 实现实时监控（<10ms 响应）——一个模型盯着干活的那个，一旦发现 sloppy、理解偏差、或"扛着红旗反红旗"，**立刻打断重来**——不等垃圾堆积成山。

## 行业在往哪里走（2026 年 6 月）

provenance-first 记忆这个方向正在收束。多个团队独立收敛到了相似的结论：

- **TierMem**（arXiv 2602.17913, 2026年2月）——provenance-aware 分层记忆，带源指针。减少 54% token，保持准确率。
- **Eywa**（arXiv 2605.30771, 2026年5月）——"证据在信念之前"，不可变源→验证→派生事实。零 LLM 检索。LoCoMo 90.19%。
- **Harness-1**（arXiv 2606.02373, 2026年6月）——外部状态 harness 让 20B 模型搜索能力超过 GPT-5.4。
- **Anthropic**——结构化信息组织让 SQL 准确率从 21% → 95%。Dreaming 跨 session 记忆巩固。
- **Mem0**——市场领导者（LongMemEval 94.8%），ADD-only 提取。
- **Palantir AIP**——provenance-aware 删除，企业本体驱动的 Agent 记忆。

我是从不同的起点独立走到这里的——不是读论文读到的，是物理 PhD 的思维训练 + 反复实验 + 摔键盘摔出来的。我觉得最少被探索、也最让我感兴趣的角度，是**主动证伪**（不止检查证据来源，还要主动搜索反例）和**正交实时监督**（另一个模型盯着干活的那个，用硬中断而不是软 prompt）。我不声称优先权。我分享的是一套想透了的取舍，和一个能跑的核心。

## 当前状态（2026年6月11日）

**能跑的：**
- L1 检索：10 通道 RRF 融合，零 LLM，ChannelRegistry 可插拔。P50 延迟 **0.1s**。向量覆盖：**96.3%**。RRF 融合 **0 锚点丢失**。
- L3 核心机制：`action_score`、`recall_counter`（证伪搜索）、`cites_layer0`（查锚）强制执行。Court 证伪 **100%** (smart)。
- **对抗安全：80%**（目标 >70%）。独有——Mem0/Letta/Zep 无等价能力。
- **A2A 成功率：100%** (20/20) — 带 provenance 链的结构化 agent 间通信。
- Agent 全栈通：session + bootstrap + prefix cache 正常，成功修改文件。
